# Citation Builder Campaign Workflow

A citation builder campaign helps a local business turn a Citation Tracker-style
report into a reviewed directory submission plan.

The goal is not to let software publish business information unchecked. The
workflow standardizes the intake, prepares submission packets, tracks claim and
verification status, and keeps final listing approval under human review.

## Trigger

The workflow starts when a customer provides:

- a Citation Tracker, BrightLocal, Whitespark, Semrush, or similar citation
  report
- a list of missing, incorrect, pending, or competitor citations
- a request to improve local SEO coverage or NAP consistency

## Customer Intake

Collect one canonical business profile before any submission work begins:

- legal or public business name
- street address or service-area preference
- zip or post code
- primary phone number
- website URL
- public email address, if used in directories
- primary and secondary categories
- short business description
- hours
- service areas
- logo and approved photos
- preferred account owner for each directory

If the business is service-area only, confirm whether the street address should
be hidden wherever the directory supports it.

## Report Normalization

Convert the source report into four working groups:

- Found and correct
- Found with NAP errors
- Not found
- Pending or needs verification

For each directory, capture:

- directory name
- report status
- listing URL, if present
- name shown
- address shown
- zip or post code shown
- phone shown
- account or login owner
- required customer action
- verification method
- next review date

## Approved AI Actions

AI can prepare:

- a normalized citation tracker
- directory-specific submission notes
- paste-ready descriptions from approved customer copy
- NAP mismatch summaries
- customer verification request drafts
- weekly status summaries
- rescan readiness notes

## Human Review Gates

A human must approve:

- the canonical NAP profile before submissions start
- any directory that requires payment
- any account login, password, or ownership change
- any decision to hide or publish a street address
- any category that could misrepresent the business
- any listing description that adds claims not supplied by the customer
- final submission or claim completion

## Reviewed Packet

Each citation packet should include:

- customer name
- canonical NAP profile
- source report date
- citation score at intake
- summary counts by status
- prioritized directory list
- paste-ready business description
- required customer verifications
- paid directory decisions
- blocked or unavailable directories
- target rescan date

## Execution Steps

1. Confirm canonical NAP and service-area rules with the customer.
2. Normalize the report into the citation tracker.
3. Mark high-trust platforms first: Google, Apple Maps, Facebook, Bing, Yelp,
   and BBB.
4. Prepare directory packets for each missing or incorrect listing.
5. Submit, claim, or update listings only after human approval.
6. Record listing URLs, account owner, verification method, and status.
7. Send the customer only the verification actions they must complete.
8. Re-check visible listings after the directory's normal review period.
9. Re-run the citation report and compare score, found count, and NAP errors.

## Status Definitions

Use consistent status labels:

- `not_started`
- `prepared`
- `submitted`
- `customer_verification_needed`
- `pending_directory_review`
- `live_correct`
- `live_with_nap_error`
- `blocked`
- `paid_decision_needed`
- `not_applicable`

## Customer-Facing Summary

TeamShift can turn a citation audit into a managed local listing campaign. The
customer provides the audit and approves the canonical business profile; TeamShift
prepares the submissions, tracks verification, flags paid or sensitive decisions,
and reports back after the listings are live or ready for rescan.
